# Documentación SportMatch

## Introducción

Este sitio contiene la documentación principal del proyecto desarrollado para la asignatura de Aplicaciones de Código Abierto. El propósito de esta documentación es describir de forma clara la selección, análisis, contribución y seguimiento del trabajo realizado sobre un proyecto de código abierto existente, utilizando herramientas como Codeberg para el control de versiones y MkDocs para la documentación técnica.

El proyecto se desarrolló siguiendo buenas prácticas de contribución a software libre, tomando en cuenta la comprensión del código base existente, la identificación de mejoras, la gestión de incidencias y la organización del equipo para realizar aportes concretos al proyecto seleccionado.

---

## Objetivo del proyecto

El objetivo principal fue contribuir de forma activa a un proyecto de código abierto real, aplicando metodologías de trabajo colaborativo, control de versiones y gestión de incidencias. Las contribuciones realizadas incluyeron la implementación de nuevas funcionalidades y la corrección de errores o bugs identificados en el sistema.

A través de este proyecto se buscó demostrar la capacidad del equipo para integrarse al ciclo de desarrollo de un proyecto existente, comprender su arquitectura, proponer mejoras justificadas y documentar el trabajo aportado.

---

## Descripción general

El proyecto seleccionado fue SportMatch, una plataforma de reserva y gestión de canchas deportivas. El equipo analizó el código base existente, identificó oportunidades de mejora y realizó contribuciones concretas organizadas mediante issues en Codeberg, donde se registró y dio seguimiento a cada tarea, corrección o funcionalidad implementada.

La documentación generada con MkDocs centraliza la información sobre las decisiones técnicas tomadas, las contribuciones realizadas y el proceso de trabajo seguido por el equipo, facilitando su consulta por parte de docentes, integrantes o futuros colaboradores del proyecto.

---

## Herramientas utilizadas

Durante el desarrollo del proyecto se utilizaron las siguientes herramientas:

| Herramienta   | Uso principal                                 |
|---------------|-----------------------------------------------|
| Codeberg      | Repositorio, issues y gestión del proyecto    |
| Git           | Control de versiones                          |

---

## Gestión del proyecto

La gestión del proyecto se realizó mediante un tablero de trabajo en Codeberg Projects.  
Este tablero permite organizar las actividades según su estado, facilitando el control del avance del proyecto.

Las tareas se clasifican normalmente en estados como:

- Bakclog
- Por hacer
- En Progreso
- Finalizado

Cada issue representa una actividad específica del proyecto, como una mejora, corrección, funcionalidad nueva o ajuste técnico.

---

## Estructura de la documentación

Esta documentación se encuentra organizada en diferentes secciones para facilitar su lectura:

- **Inicio:** presentación general del proyecto.
- **Requerimientos:** descripción de las necesidades funcionales y no funcionales.
- **Gestión de issues:** explicación de las tareas registradas en Codeberg.
- **Metodología:** forma de trabajo utilizada por el equipo.
- **Instalación:** pasos necesarios para ejecutar el proyecto.
- **Pruebas:** validaciones realizadas sobre el sistema.
- **Conclusiones:** resultados obtenidos y aprendizajes del proyecto.

---

## Importancia de la documentación

La documentación es una parte fundamental del desarrollo de software, ya que permite comprender el funcionamiento del sistema, justificar las decisiones tomadas y facilitar el mantenimiento futuro.

Además, ayuda a evidenciar el proceso seguido por el equipo, mostrando no solo el producto final, sino también la forma en que se organizó, planificó y desarrolló el proyecto.

---

## Conclusión

Este proyecto representa la aplicación práctica de conceptos de Ingeniería de Software, integrando planificación, gestión de tareas, desarrollo colaborativo, control de versiones y documentación técnica.

El uso de Codeberg y MkDocs permite mantener un proceso ordenado, transparente y fácil de consultar, fortaleciendo la calidad del proyecto y su presentación académica.
